Key behaviours

  • Drag from tray → canvas — widget pre-configured with field data, drops into nearest valid zone or between blocks
  • Drag within canvas — reorder within a block or move between blocks; drop-between-blocks zone creates a new section
  • Widget suggestion — driven by existing fieldRole + fieldType already in the registry. No new logic needed.
  • Ignored fields — shown greyed out, not draggable. Prevents confusion.
  • Widgets tab — blank widget types, identical to current quick-add palette
  • Templates tab — pre-built layouts (feeds Phase 2)
  • Tray source — shows which import batch the data fields come from; user can switch if multiple imports exist for the period

What stays the same

Story blocks, zone keys, sort order, widget instances, approval/versioning, snapshot model — all unchanged. Drag-and-drop is a UI layer on top of the existing saveWidgetInstance / deleteWidgetInstance actions.
